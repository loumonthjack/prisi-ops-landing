### Site Speed Optimization

**Current Targets**:
- First Contentful Paint (FCP): < 2s â±ï¸
- Largest Contentful Paint (LCP): < 2.5s â±ï¸
- First Input Delay (FID): < 100ms âš¡
- Cumulative Layout Shift (CLS): < 0.1 âœ…

**Optimization Tactics**:

1. **Minify CSS/JS**:
   - Use Vite build optimization
   - Remove unused CSS with PurgeCSS
   - Tree-shake JavaScript imports

2. **Enable Compression**:
   - Gzip compression on server
   - Brotli compression (better than Gzip)
   - Compress all text assets (HTML, CSS, JS, JSON)

3. **Use CDN**:
   - Serve static assets from CDN (Cloudflare, AWS CloudFront)
   - Reduce latency with edge locations
   - Automatic image optimization

4. **Code Splitting**:
   - Lazy load routes (React Router lazy loading)
   - Dynamic imports for non-critical components
   - Defer non-essential JavaScript

5. **Optimize Images**:
   - Use modern formats (WebP, AVIF)
   - Implement lazy loading
   - Set explicit dimensions
   - Use appropriate sizes (don't load 4K on mobile)

### Mobile Optimization

**Google uses mobile-first indexing** - mobile version is primary.

**Requirements**:
- âœ… Mobile-responsive design
- âœ… Touch targets minimum 44x44px
- âœ… Readable font sizes (16px base)
- âœ… No horizontal scroll
- âœ… Fast mobile load times

**Testing**:
- Test on real devices (iOS Safari, Chrome Android)
- Use Chrome DevTools mobile emulation
- Test various screen sizes (320px to 428px)
- Check tap target sizes
- Verify forms work well on mobile

### Core Web Vitals

These are Google's official ranking factors:

**1. Largest Contentful Paint (LCP)** - Loading Performance
- **Target**: < 2.5 seconds (green)
- **Measures**: Time for largest element to load
- **Optimizations**:
  - Optimize hero image
  - Preload critical resources
  - Use CDN for faster delivery
  - Minimize render-blocking resources

**2. First Input Delay (FID)** - Interactivity
- **Target**: < 100ms (green)
- **Measures**: Time from first user interaction to browser response
- **Optimizations**:
  - Minimize JavaScript execution time
  - Break up long tasks
  - Use web workers for heavy computation
  - Defer non-essential JavaScript

**3. Cumulative Layout Shift (CLS)** - Visual Stability
- **Target**: < 0.1 (green)
- **Measures**: Unexpected layout shifts
- **Optimizations**:
  - Set explicit dimensions for images/videos
  - Reserve space for ads/embeds
  - Don't inject content above existing content
  - Use transform animations instead of layout changes

**Monitoring**:
- Google PageSpeed Insights
- Chrome DevTools Lighthouse
- Google Search Console Core Web Vitals report
- Real User Monitoring (RUM) with tools like Sentry
